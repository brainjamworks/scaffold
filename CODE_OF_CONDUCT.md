# Code of Conduct

Scaffold uses a concise contributor code of conduct for all project spaces:
issues, pull requests, discussions, reviews, and community channels.

This code applies to project spaces operated by Scaffold and to anyone
representing the project publicly. The project maintainer is Rizvan Ali.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent, but accept corrections when impact says otherwise.
- Keep technical disagreement focused on the work.
- Give contributors room to learn without lowering the quality bar.
- Respect privacy and avoid sharing private communications or data without
  consent.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Personal attacks or sustained disruption.
- Sexualized language or attention in project spaces.
- Publishing private information without permission.
- Retaliation against someone who reports a concern.

## Reporting

Report code-of-conduct concerns privately to
[support@scaffold.ac](mailto:support@scaffold.ac). Please include the
relevant project space, what happened, and any immediate safety concern. If the
concern involves a security issue, follow [SECURITY.md](./SECURITY.md) instead.

Maintainers may remove comments, close threads, block accounts, or take other
reasonable action to protect the project community.
