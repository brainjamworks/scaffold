# Security Policy

Scaffold does not have a formal public security program or SLA yet.

The current release is pre-alpha and does not promise a supported-version
window. Security reports are still welcome and will be triaged by the project
maintainer.

Please do not open a public issue for a suspected vulnerability. Report it
privately through GitHub private vulnerability reporting if it is enabled for
this repository. If that is unavailable, email
[support@scaffold.ac](mailto:support@scaffold.ac) with the subject
`Security report: Scaffold` before publishing details.

Include:

- Affected package, adapter, or command.
- Steps to reproduce.
- Expected impact.
- Any known workaround.

Avoid sending real credentials, learner data, or production LMS data in a
report. Use minimal reproduction data whenever possible.

The maintainer is Rizvan Ali.
