# Support

Scaffold is pre-alpha software. APIs, adapter contracts, document details, and
supported host versions may change before `1.0`.

## Contact

For general questions, installation problems, documentation issues, and
responsible project feedback, email
[support@scaffold.ac](mailto:support@scaffold.ac).

Do not include passwords, access tokens, learner records, or production LMS data
in support requests.

## Before Contacting Support

- Check the root README and the relevant package or adapter README.
- Include the Scaffold version or commit, host platform and version, operating
  system, and the smallest reproduction you can share.
- Include the exact command run and the complete relevant error output.
- For security vulnerabilities, follow [SECURITY.md](./SECURITY.md) instead.

## Issues and Contributions

Use GitHub issues for reproducible bugs and scoped feature requests. Use pull
requests for proposed changes. Please read [CONTRIBUTING.md](./CONTRIBUTING.md)
before contributing.
